## Changelog

### Improvements

- **REST Connector:** You can now ingest metadata from a local JSON/CSV file via the REST connector instead of only a remote URL (#26036).
- **Snowflake:** Ingestion no longer fetches stage definitions, reducing unnecessary API calls and improving performance (#26009).
- **UI - Test Cases:** Test case summary list loads and renders faster (#26004).

### Bug Fixes

- **UI - Data Product:** The Data Product description area now scrolls correctly when content is long (#26040).
- **UI - Tags:** Pressing Enter in a tag input no longer clears tags you had already added (#26039).
- **UI - Translation:** Global Domain Filter label and options now show the correct translated text (#25982).
- **UI - Translation:** User dropdown in the header now displays translated strings correctly (8baf4897fe).
- **UI - Data Product:** When validation fails on the Data Product form, the page scrolls to the error so it is visible (#25760).
- **Ingestion Pipeline:** PATCH requests to prepare a pipeline for deploy no longer fail due to a missing service reference (#25984).
- **Data Quality:** Completing a test suite pipeline now correctly triggers ChangeEvents for related test suites (#25910).
- **Search:** Domain filters in search and discovery now return the correct results from Elasticsearch (#26037).
- **Database - GET_DDL:** Table DDL with special characters in names (e.g. spaces, reserved words) is now generated correctly (#26007).
- **Ingestion:** Pipelines no longer fail with ValidationError when using Pydantic v2–compatible configs (#26014).
- **dbt:** When multiple `run_results` entries exist for the same unique_id, the latest one is used so lineage and tests stay in sync (#25996).
- **Datalake Connector:** Resolved connector errors so Datalake ingestion runs reliably (#25953).
